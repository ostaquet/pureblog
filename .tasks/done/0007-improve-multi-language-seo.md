# Problem to solve

The URL of a post is based on a slug. The slug is defined by the filename of the Markdown.

Typically, I have a Markdown file `hello-world.en.md` which is served by the URL `en/hello-world/`.

The URL is critical to offer proper SEO visibility. At the moment, the slug for the 3 languages must be the same to switch the language on a page. Example: `en/hello-world/` for the English version and the `fr/hello-world/` and `nl/hello-world` for the others.

This is not working as I would like to have a different URL per language for the same post to improve SEO.
